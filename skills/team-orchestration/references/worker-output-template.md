# Worker Output Template

Use this when a worker needs a compact output shape that the conductor can persist directly.

```md
# What I Inspected
- file or surface
- file or surface

# Concrete Findings Or Returned Work
- finding or result
- finding or result

# Unresolved Questions
- unresolved question
- unresolved question

# Confidence
- High | Medium | Low
- Why
```

Keep the output evidence-first. If a claim depends on runtime behavior or a third-party system that was not exercised, mark it clearly as `UNVERIFIED`.
